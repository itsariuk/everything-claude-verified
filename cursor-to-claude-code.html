two things to know before you switch

What carries over. What doesn't.

carries over

.cursorrules → CLAUDE.md

Your Cursor rules file is a direct equivalent. Copy it to CLAUDE.md at the project root. Claude Code loads it on every session automatically.

cp .cursorrules CLAUDE.md
doesn't carry over

No codebase index.

Cursor has semantic vector search across your repo. Claude Code doesn't — it reads files explicitly via tools. Use @filename to attach context, or ask the agent to explore first.

"explore the billing module before editing"

16 / 34
why isolation matters · the science

Context is finite. Pollution is real.

Lost in the Middle · Stanford 2023

LLMs perform significantly worse on information placed in the middle of long contexts. Relevant facts get buried. A reviewer who watched you debug for 2 hours has worse signal than one who sees only the diff.

context pollution

Irrelevant tokens compete with task-relevant ones for model attention. Every prior failed attempt, tangential discussion, and re-explained requirement degrades the quality of the next response.

context grounding

Grounding is the mechanism by which a model anchors its output to the provided context rather than to its training weights. A grounded model generates facts from what it can see — not from what it was taught. Focused, task-specific context maximises grounding: fewer invented facts, higher factual accuracy, outputs that are actually about your code.

the isolated agent advantage

Each specialist starts clean.

The code-reviewer sees only the diff.
The security-reviewer sees only the changed files.
The planner sees only the requirements.

No shared baggage. No accumulated bias. No middle-of-context blind spots.

why this is one of your highest-leverage tools

You can't make the model smarter. You can control what it sees. Isolated context is the architectural lever that makes every agent sharper — without touching a single model weight.

hooks

Lifecycle enforcement. Not guidance.

Hooks run in the harness, not the model. The agent can't forget them, skip them, or talk its way out of them. They're the reason "please remember to" prompts stop being necessary.
PreToolUse

Before it happens

Validate, redact, or block. Secret scan before commit. Permission gate before rm -rf.

PostToolUse

After it happens

Format, lint, index. Prettier on the diff. Re-run the type checker. Write to the codemap.

Stop

At the end

Final verification. Run the test gate, write a session summary, checkpoint state — whatever you wire in.

29 / 34
context is a resource

1M context. Still worth being surgical.

Every enabled MCP loads tools into the active window. More tools = more noise, slower turns, higher cost. The window is large — don't fill it with tools you're not using.
the problem

Tool bloat

20+ MCPs all enabled = 80+ active tools loading on every turn. More noise, slower inference, higher cost per message.

the rule

20–30 configured.
Under 10 active.

Keep a full roster in config. Enable only what today's task needs. Disable the rest per project.

compact

Compact strategically.

Auto-compact hits at arbitrary points. Use /compact at phase boundaries — after exploration, before execution.
